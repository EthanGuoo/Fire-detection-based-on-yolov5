# 依赖修复说明

> 💡 **提示**: 所有脚本已配置清华镜像源，下载速度更快！

## 问题描述

如果运行 `python wzq.py` 时出现以下错误：

```
A module that was compiled using NumPy 1.x cannot be run in NumPy 2.4.4...
AttributeError: module 'shiboken6.Shiboken' has no attribute 'replaceModuleDict'
```

这是因为 **NumPy 2.x 与旧版本模块不兼容**。

---

## 快速修复（推荐）⭐

> ✅ **已使用清华镜像源，下载速度更快！**

### Windows PowerShell:

1. **右键点击** `fix_dependencies.ps1`
2. 选择 **"使用 PowerShell 运行"**

或在 PowerShell 中执行：
```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\fix_dependencies.ps1
```

### 如果遇到执行策略限制：

```powershell
# 临时允许执行脚本
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

# 然后运行修复脚本
.\fix_dependencies.ps1
```

## 手动修复

如果自动脚本不工作，请在 PowerShell 或命令提示符中**依次**执行：

```bash
# 1. 卸载不兼容的 NumPy 2.x
pip uninstall numpy -y

# 2. 安装兼容的 NumPy 1.x（使用清华镜像源）
pip install "numpy>=1.18.5,<2.0.0" -i https://pypi.tuna.tsinghua.edu.cn/simple

# 3. 重新安装 PySide6（使用清华镜像源）
pip uninstall PySide6 shiboken6 -y
pip install "PySide6>=6.8.0" -i https://pypi.tuna.tsinghua.edu.cn/simple

# 4. 验证安装
python -c "import numpy; import PySide6; print(f'NumPy: {numpy.__version__}'); print(f'PySide6: {PySide6.__version__}')"
```

## 常见问题

### Q: PowerShell 无法执行脚本？
**A:** 运行 `Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass`

### Q: pip 命令找不到？
**A:** 确保 Python 已添加到系统 PATH，或使用完整路径：
```bash
python -m pip install "numpy<2.0.0"
```

### Q: 修复后仍然报错？
**A:** 尝试：
1. 重启命令行窗口
2. 检查是否有多个 Python 环境
3. 使用虚拟环境：
   ```bash
   python -m venv venv
   venv\Scripts\activate
   pip install -r requirements.txt
   ```
